# Rust Rollback Core for Godot

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Godot 4.x](https://img.shields.io/badge/Godot-4.x-blue.svg)](https://godotengine.org/)

A hyper-specialized, pure-Rust rollback netcode library built as a Godot Extension (`gdext`). Designed specifically for strict, 60-FPS 2D fighting games.

## Philosophy: Simulation vs. Presentation

This architecture strictly separates the game state from the engine:
* **The Rust Core (Simulation):** Handles 100% of the deterministic fighting game math. It uses bitmask inputs, strict Enums for character states, and fixed-point integer math for AABB hitboxes. It knows nothing about graphics or audio.
* **The Godot Engine (Presentation):** Acts as a pure renderer. Every frame, Godot polls the Rust core for coordinates and actions, drawing sprites and playing animations accordingly.

Because the entire game state is a tightly-packed, `Copy`-able Rust struct, rollbacks happen in nanoseconds without the overhead of Godot's Node system or Variant serialization.

## Building from Source

### Prerequisites
* [Rust](https://www.rust-lang.org/tools/install) (Edition 2024)
* Godot 4.x

### Compilation
To achieve the strict 16.67ms frame budget required for fighting games, this project must be compiled with extreme optimizations. 

```bash
cargo build --release
```

#### Build Output
Project is a dynamic library so the output generated by the Cargo file will be in ``target/release/`` folder.

The exact file generated depends on your operating system:

- macOS: ```librust_rollback.dylib```

- Windows: ```rust_rollback.dll```

- Linux: ```librust_rollback.so```

#### Headless Testing
Test the library without needing Godot by running:
``` bash
 cargo test
```
